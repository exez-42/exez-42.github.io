Datenmodelle im Überblick

1. Relationales Datenmodell (1970er)

  • Konzept: Daten in Tabellen (Relationen)
  • Elemente: Tabellen, Zeilen (Datensätze), Spalten (Attribute)
  • Schlüssel: Primär- und Fremdschlüssel
  • Wichtig: Normalisierung
  • DBMS: Oracle, MySQL, PostgreSQL, MS SQL Server

2. Objektorientiertes Datenmodell (1980er)

  • Konzept: Daten als Objekte
  • Elemente: Objekte, Klassen, Methoden
  • Prinzipien: Vererbung, Kapselung, Polymorphismus
  • DBMS: db4o, ObjectDB, Versant Object Database

3. Objektrelationales Datenmodell (1990er)

  • Konzept: Kombination relational und objektorientiert
  • Erweiterungen: Benutzerdefinierte Datentypen, Vererbung in Tabellen
  • DBMS: Oracle (mit Erweiterungen), PostgreSQL, IBM DB2
